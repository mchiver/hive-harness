# HiveJS: Marketing Material Roadmap

## Proposed Additional Documents

### 1. The "One-Pager" (Executive Summary)
- **Goal:** A high-impact, single-page PDF for quick sharing.
- **Content:** Value proposition, 3 core pillars (Isolation, Plugins, AI-Ready), and a "How it Works" diagram.

### 2. Feature Comparison Matrix
- **Goal:** Contrast HiveJS with traditional AI agent frameworks (e.g., LangChain, AutoGPT).
- **Key Dimensions:** Data isolation, Auth/Security, Local vs Cloud, Plugin Ease-of-Use.

### 3. "Building Your First Plugin" Case Study
- **Goal:** Demonstrate the ease of extensibility.
- **Content:** A step-by-step guide showing a "Real World" tool (e.g., a Jira Ticket Creator) being integrated into HiveJS.

### 4. Architecture Visuals (Diagrams)
- **Registry vs Hive:** A visual representation of the global registry and the per-folder hive instances.
- **Tool Invocation Flow:** User $\rightarrow$ AI Agent $\rightarrow$ HiveJS $\rightarrow$ Plugin $\rightarrow$ Local System.

## Implementation Priority
1. **One-Pager** (Crucial for initial outreach)
2. **Architecture Visuals** (Essential for technical understanding)
3. **Comparison Matrix** (Necessary for competitive positioning)
4. **Plugin Case Study** (Important for developer adoption)
